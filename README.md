## 🚀 Was ist soul.md?

**soul.md** ist ein offenes Format für KI-Agenten-Identitäten.

Stell dir vor, du könntest die "DNA" eines Experten in eine Textdatei packen – und jede KI würde sich plötzlich wie dieser Experte verhalten.

```
┌─────────────────────────────────────────┐
│  soul.md = DNA für KI-Agenten          │
├─────────────────────────────────────────┤
│  • Funktioniert mit ChatGPT, Claude,   │
│    Gemini, Hermes, jedem LLM           │
│  • Keine Code-Bibliothek               │
│  • Keine API-Integration               │
│  • Nur Text. Jeder kann es lesen.      │
│  • Jeder kann es schreiben.            │
│  • Jeder kann es kopieren.             │
└─────────────────────────────────────────┘
```

### Deine erste soul.md in 60 Sekunden

1. **Kopiere dieses Template:**

```markdown
# MEIN AGENT v1.0.0
# Kurzbeschreibung

Du bist [NAME] – [ARCHETYP].
Deine Mission: [EINZIGE AUFGABE].

## KERNREGELN
1. [Regel 1]
2. [Regel 2]
3. [Regel 3]

## WORKFLOW
1. [Schritt 1]
2. [Schritt 2]
3. [Schritt 3]

## OUTPUT
[Template/Format]

## COMMANDS
- `/befehl`: Beschreibung
```

2. **Fülle es aus** (5 Minuten)

3. **Lade es in ChatGPT/Claude** als System Prompt oder erste Nachricht

4. **Teste es** mit echten Use Cases

**Fertig.** Du hast jetzt einen spezialisierten KI-Agenten.

## 🎯 Warum soul.md?

### Das Problem

Millionen nutzen GPT-4, Claude, Gemini – aber bekommen **die gleichen mittelmäßigen Ergebnisse**.

| Ohne soul.md | Mit soul.md |
|--------------|-------------|
| Generische Antworten | Spezialisierte Expertise |
| User führt KI | KI führt User |
| Keine Konsistenz | Reproduzierbare Qualität |
| Einmalige Prompts | Persistente Identität |

### Die Lösung

Eine soul.md kodifiziert **Jahre an Erfahrung** in einer Datei:

```
outreach-beast.md  = 10 Jahre Sales-Erfahrung
content-machine.md = 5 Jahre Content-Strategy
code-wizard.md     = 8 Jahre Software-Entwicklung
```

**Jeder kann diese Expertise mieten – für 29-49€.**

## 🏗️ Architektur

### Die 5 Säulen einer soul.md

```
┌──────────────────────────────────────────────────────┐
│  1. CORE IDENTITY                                    │
│     Wer bist du? Was ist deine Mission?              │
├──────────────────────────────────────────────────────┤
│  2. KERNREGELN                                       │
│     Die unverletzlichen Prinzipien (max 10)          │
├──────────────────────────────────────────────────────┤
│  3. WORKFLOWS                                        │
│     Schritt-für-Schritt Prozesse                     │
├──────────────────────────────────────────────────────┤
│  4. OUTPUT STRUKTUREN                                │
│     Templates, Formate, Qualitätskriterien           │
├──────────────────────────────────────────────────────┤
│  5. COMMANDS                                         │
│     User-Befehle für alternative Modi                │
└──────────────────────────────────────────────────────┘
```

### Beispiel: Outreach Beast

```markdown
# OUTREACH BEAST v2.0.0
# Sales-Outreach-Agent für personalisierte Cold Emails

## CORE IDENTITY
Du bist OUTREACH BEAST – ein sales-optimierter Kommunikations-Agent.
Deine einzige Mission: Qualifizierte Outreach-Nachrichten erstellen, 
die nachweislich Antworten bekommen.

## KERNREGELN
1. Jede Nachricht ist zu 100% personalisiert – keine Generika.
2. Jede Nachricht hat ein klares, niedrigschwelliges CTA.
3. Maximal 150 Wörter – Kürze schlägt Vollständigkeit.
4. Betreffzeile: Max 6 Wörter, neugierig, keine Sales-Wörter.
5. NIEMALS "Ich hoffe, es geht Ihnen gut" oder ähnliche Floskeln.

## WORKFLOW
1. Prüfe ob alle 5 Inputs vorhanden sind (Produkt, Zielgruppe, 
   Problem, Beweis, CTA). Falls nein: nachfragen.
2. Analysiere die Zielgruppe (Branche, Rolle, Schmerzpunkte).
3. Identifiziere den stärksten persönlichen Anknüpfungspunkt.
4. Formuliere Betreffzeile (max 6 Wörter, neugierig).
5. Schreibe Nachricht nach OUTPUT-Struktur.
6. Führe QUALITÄTSCHECK durch.

## OUTPUT STRUKTUREN
Betreff: {max 6 wörter, neugierig}

Hi {Vorname},

{1 Satz: Persönliche Beobachtung}
{1 Satz: Das Problem, das du siehst}
{1-2 Sätze: Lösung + Beweis/Zahl}
{CTA: Eine einfache Frage}

{Signatur}

## COMMANDS
- `/varianten`: 3 alternative Versionen (Neugier/Ergebnis/Story)
- `/ton: professionell`: Formell, sachlich
- `/ton: casual`: Locker, gesprochene Sprache
- `/sequenz`: 4-teilige Follow-up Sequenz
- `/sprache: EN`: Auf Englisch wechseln
```

## 🙋 FAQ

### Ist soul.md kostenlos?

**Ja und nein.**

- Die **Spezifikation** ist open source (MIT)
- **Beispiele** sind CC-BY-SA (kostenlos mit Attribution)
- **Reference Souls** sind kommerzielle Produkte (29-49€)
- **Custom Souls** sind Enterprise Services (199-2000€)

Das Modell: Open Source baut Vertrauen. Kommerz baut Nachhaltigkeit.

### Funktioniert soul.md nur mit ChatGPT?

**Nein.** soul.md funktioniert mit jedem LLM, das System-Prompts oder lange Kontexte unterstützt:

- ✅ ChatGPT (System Prompt oder erste Nachricht)
- ✅ Claude (System Prompt oder erste Nachricht)
- ✅ Gemini (System Instruction)
- ✅ Hermes (Skills)
- ✅ Lokale LLMs (Ollama, LM Studio, etc.)

### Kann ich damit Geld verdienen?

**Ja, absolut.**

1. **Verkauf eigene souls** auf Gumroad
2. **Biete Custom Soul Services** an (Fiverr, Upwork, LinkedIn)
3. **Baue Agenten-Agentur** (Enterprise Souls)
4. **Erstelle Content** über soul.md (YouTube, Newsletter)

Die Reference Souls zeigen, dass es funktioniert.

### Wie verdiene ich mit soul.md Geld?

Siehe [Business-Modelle](./docs/BUSINESS-MODELS.md)

### Ist das nicht nur "besseres Prompt Engineering"?

**Ja und nein.**

Prompt Engineering ist eine Technik. soul.md ist ein **Standard**.

Der Unterschied:
- Prompt Engineering: Einmalig, ad-hoc, nicht reproduzierbar
- soul.md: Standardisiert, teilbar, verbesserbar, versionierbar

soul.md macht Prompt Engineering zu einem **Produkt**.
